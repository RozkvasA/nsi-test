# Этап 12: упрощение отображения данных и режимы чтения

Этот документ применяется вместе с:

```text
docs/NSI_MODULE_TARGET_SPEC.md
docs/NSI_OVERVIEW_USABILITY_FIX.md
docs/NSI_STAGE10_ENGINEERING_STRUCTURE_AND_CARD_UX.md
docs/NSI_STAGE11_OBJECT_TREE_ENGINEERING_VIEW.md
```

## 1. Назначение этапа

Этап 12 нужен не для добавления новых сущностей, а для снижения визуальной перегрузки интерфейса.

Текущая модель становится логически правильнее: объект имеет Помещения и Системы, система является инженерным контуром, оборудование является физической единицей размещения.

Но интерфейс не должен заставлять пользователя видеть всю сложность модели одновременно.

Цель: сделать отображение данных спокойным, очевидным и практичным.

## 2. Практическая проблема

Пользователь не должен ощущать, что дерево объектов это перегруженная техническая схема.

Если в одной строке или на одном экране одновременно показывать:

```text
помещения
системы
наследуемые системы
оборудование
состав оборудования
предупреждения
служебные признаки
```

интерфейс становится тяжелым и неочевидным.

Правильный подход: сначала показывать структуру, а детали раскрывать постепенно.

## 3. Принцип прогрессивного раскрытия

По умолчанию дерево должно показывать только навигационный минимум:

```text
название
тип коротко
количество вложенных помещений
количество систем
критичное предупреждение, если есть
```

Остальное показывать только после раскрытия соответствующей папки или карточки.

Нельзя в каждой строке постоянно показывать все зависимости.

## 4. Режимы отображения дерева

Для Дерева объектов нужны два режима отображения:

```text
Компактный
Подробный
```

Компактный режим используется по умолчанию.

В компактном режиме:

```text
показывать короткие строки
не показывать длинные summary
не показывать состав оборудования до раскрытия системы
не показывать полный список наследуемых систем в каждой дочерней папке
показывать только счетчик наследуемых систем или короткий бейдж
```

Подробный режим:

```text
показывать больше счетчиков
показывать наследуемые системы в списке
показывать больше диагностик
показывать состав инженерной структуры глубже
```

Если нет времени делать переключатель, нужно хотя бы сделать поведение как компактный режим по умолчанию.

## 5. Системы в дереве: не перегружать дочерние уровни

Наследуемые системы не должны превращать каждую дочернюю папку Системы в огромный список одинаковых элементов.

В компактном режиме для дочернего объекта показывать так:

```text
Системы · сист. 0 · наслед. 4
```

или внутри папки:

```text
наследуется 4 системы
```

Полный список наследуемых систем показывать только по раскрытию или в подробном режиме.

Собственные системы показывать всегда, потому что они заведены именно на этом уровне.

## 6. Система как контур, оборудование как наполнение

В дереве система должна выглядеть как контур, а не как физическая единица.

Строка системы должна показывать:

```text
название системы
область действия коротко
количество оборудования
предупреждение нет оборудования, если пустая
```

Не показывать в строке системы:

```text
место размещения
placementObjectId
длинный список помещений
длинный список оборудования
```

Оборудование показывать только внутри раскрытой системы или группы Самостоятельное оборудование.

## 7. Оборудование: показывать только там, где пользователь ожидает

Оборудование не должно засорять уровни помещения, если пользователь просто смотрит структуру помещений.

Правильные места для оборудования:

```text
внутри раскрытой системы
внутри группы Самостоятельное оборудование
в карточке помещения в таблице Оборудование в помещении
в карточке оборудования
```

Нельзя дублировать одно и то же оборудование слишком много раз в дереве.

Если оборудование отображается как часть системы, в помещении его достаточно показывать в карточке помещения, а не обязательно раздувать дерево помещений.

## 8. Папки Системы и Помещения

Папки Системы и Помещения должны быть понятными и спокойными.

Папка Помещения:

```text
показывает вложенные помещения и объекты размещения
не показывает оборудование прямо в строке
```

Папка Системы:

```text
показывает собственные системы
показывает краткий признак наследуемых систем
показывает Самостоятельное оборудование отдельной группой только если оно есть
```

Если папка пустая, показывать коротко:

```text
нет помещений
нет систем
```

## 9. Карточка как место деталей

Дерево должно быть навигатором, а не местом полного анализа.

Подробности должны уходить в правую карточку:

```text
список оборудования помещения
полный список наследуемых систем
детали области действия системы
состав оборудования
служебные идентификаторы
диагностики
```

В дереве оставлять только короткие признаки и счетчики.

## 10. Визуальная плотность

Интерфейс должен быть менее шумным:

```text
меньше текста в строках
меньше бейджей одновременно
меньше повторяющихся подписей
меньше ярких цветов
больше воздуха между смысловыми группами
```

При этом строки дерева не должны стать огромными карточками.

Нужен баланс: компактная строка + понятная иерархия.

## 11. Адаптивность

На большом мониторе можно показывать дерево и карточку рядом.

На ноутбуке:

```text
дерево уже
карточка шире
строки дерева максимально компактные
без горизонтального скролла всей страницы
```

На iPad/tablet:

```text
дерево и карточка не съезжают
допускается вертикальная компоновка дерево сверху, карточка ниже
действия не должны вылезать за экран
длинные summary скрываются или сокращаются
```

## 12. Что не делать

Нельзя:

```text
добавлять новые сущности без необходимости
возвращать плоское дерево без папок Системы / Помещения
дублировать наследуемые системы как реальные системы дочернего объекта
показывать оборудование одновременно везде
показывать систему как физическое место
заставлять пользователя читать длинные строки
ломать Обзор объектов
```

## 13. Проверка этапа

После реализации проверить:

```text
1. Обзор объектов не изменился.
2. Дерево объектов стало визуально проще.
3. В компактном режиме строки дерева короткие.
4. Нет ощущения, что в каждой строке показана вся база данных.
5. Наследуемые системы не раздувают дочерние уровни.
6. Система показывает область действия и количество оборудования.
7. Пустая система явно помечена как нет оборудования.
8. Оборудование видно внутри раскрытой системы или в карточке помещения.
9. Самостоятельное оборудование видно отдельной группой только если оно есть.
10. Длинные списки и диагностики перенесены в карточки.
11. На ноутбуке нет горизонтального скролла всей страницы.
12. На iPad/tablet интерфейс не разваливается.
13. Пользователь может понять структуру БЦ за 10 секунд без чтения длинных подписей.
```
