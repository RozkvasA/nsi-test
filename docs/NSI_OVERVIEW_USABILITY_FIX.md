# Дополнение к ТЗ: правки Обзора объектов

Этот файл уточняет требования к начальному экрану Обзор объектов и применяется вместе с docs/NSI_MODULE_TARGET_SPEC.md.

## 1. Прокрутка Обзора объектов

При раскрытии корневого объекта, этажа или внутреннего узла пользователь должен иметь возможность прокручивать содержимое Обзора объектов.

Нельзя блокировать прокрутку из-за overflow: hidden на контейнерах Обзора объектов, work-area, overview-panel или overview-list.

Правило реализации:

```text
app-shell / app-body могут фиксировать общий каркас
рабочая область Обзора объектов должна иметь min-height: 0
overview-panel или overview-list должны иметь overflow: auto
раскрытие этажей не должно делать страницу недоступной для скролла
```

Если выбран отдельный внутренний скролл панели, он должен быть очевиден и работать колесом мыши, трекпадом и жестом на iPad.

## 2. Добавление нового корневого объекта из Обзора

Кнопка Добавить объект в Обзоре объектов должна создавать именно корневой объект, например новый БЦ, новый склад, новый отель или новый производственный объект.

Нельзя создавать новый объект как дочерний элемент уже выбранного БЦ Lucky только потому, что Lucky был текущим selectedRef.

Правило поведения:

```text
Обзор объектов -> Добавить объект
переключает раздел на Дерево объектов
открывает черновик rootObject
parentObjectId = null
creationMode = empty
```

Кнопка Создать из шаблона в Обзоре объектов должна делать то же самое, но с creationMode = template.

## 3. Раскрытие этажей и внутренних объектов

В Обзоре объектов все уровни должны быть изначально свернуты.

Когда пользователь раскрывает корневой объект, он видит этажи или крупные уровни.

Когда пользователь раскрывает этаж, внутренние объекты этого этажа тоже должны сначала отображаться свернутыми строками со сводной информацией, а не сразу раскрывать блоки помещений и систем.

В строке внутреннего объекта уровня детализации показывать компактную сводку:

```text
название
вид объекта
площадь, если есть
количество помещений
количество систем
количество оборудования
количество техкарт
предупреждения, если есть
```

Только после раскрытия внутреннего объекта показывать компактные блоки:

```text
Помещения
Системы
Оборудование, если есть
Техкарты, если есть
```

Полная глубина помещений ниже уровня детализации в Обзоре объектов не раскрывается. Для полной структуры пользователь переходит в Дерево объектов.

## 4. Компактность интерфейса

Обзор объектов должен оставаться плотным экраном в стилистике АСКЭ.

Нельзя возвращать большие карточки метрик, длинные описательные тексты и рекламные заголовки.

Допускается:

```text
короткий заголовок
компактная панель действий
строки дерева
бейджи со счетчиками
тонкие разделители
мягкая подсветка раскрытых строк
```

## 5. Проверка после правки

После доработки необходимо проверить:

```text
1. Обзор объектов скроллится после раскрытия этажей.
2. Добавить объект из Обзора создает новый корневой объект, а не дочерний элемент БЦ Lucky.
3. Создать из шаблона из Обзора создает новый корневой объект по шаблону.
4. Lucky изначально свернут.
5. Этажи изначально свернуты.
6. Внутренние объекты этажа изначально свернуты.
7. Блоки Помещения и Системы появляются только после раскрытия внутреннего объекта.
8. Правая карточка в Обзоре объектов не появляется.
```
