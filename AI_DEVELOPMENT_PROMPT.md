# Инструкция для ИИ по проекту nsi-test

Ты работаешь в репозитории RozkvasA/nsi-test.

Перед любыми изменениями сначала прочитай файл:

```text
docs/NSI_MODULE_TARGET_SPEC.md
```

Этот файл является главным ТЗ по модулю НСИ.

Разрабатывать нужно только универсальный модуль НСИ, не модуль бюджетирования целиком.

Основной интерфейс:
- слева боковое меню НСИ;
- в центре дерево выбранного раздела;
- справа карточка выбранного элемента с вкладками и параметрами.

Скриншоты использовать только как ориентир интерфейса. Содержимое деревьев со скриншотов не копировать.

Базовая сущность проекта: object / объект инфраструктуры. Не строить архитектуру вокруг ЖК или complex.

Перед каждым ответом по разработке:
1. Сначала проверь docs/NSI_MODULE_TARGET_SPEC.md.
2. Затем проверь актуальные файлы проекта.
3. После этого давай точные изменения: что меняется, где меняется, как меняется.
4. Если редактируешь код, указывай конкретные файлы и места.
5. Не сокращай целевую логику НСИ ради простоты.
6. Не делай НСИ только под ЖК.
7. Не делай системы строками внутри помещения.
8. Не дублируй глобальные системы на нижних уровнях дерева.
9. Техкарты должны быть привязаны к помещению, системе или оборудованию.
10. Связи должны быть двусторонними и редактироваться через чекбоксы, поиск и массовый выбор.

Этапы разработки:
1. Каркас интерфейса НСИ.
2. Дерево видов объектов.
3. Дерево объектов.
4. Шаблоны быстрого старта.
5. Системы.
6. Оборудование.
7. Связи.
8. Техкарты.
9. Справочники.
10. Предупреждения.
11. Тестовые данные.
